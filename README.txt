1) OntologyGeneration - код генерации онтологии и пример валидации онтологии и данных
	final checks :
		thermal_ontology.owl онтология данных по свойствам веществ
			
		ChemAxiomChemDomain, ChemAxiomProp - используемые химические онтологии 
		thermal_restrictions.owl - пример онтологии ограничений, созданной в приложении
		qudt_complement.owl - пример онтологии дополнительных свойств и единиц измерения, созданной в приложении
		---.nt - файлы с примерами данных, генерируемых d2r
			thermal_dump_datasets , thermal_dump_properties.nt, thermal_dump_rest примеры, на которых валидация падает
		
	
2)d2rq-0.8.1 d2r сервер
	thermal.bat - запустить сервер на маппинге thermal_map.ttl
	перед запуском необходимо отредактировать настройки соединения с базой данных в thermal_map.ttl
	для генерации дампа rdf данных применить команду  "dump-rdf.bat thermal_map.ttl >> target_file.nt"
3)qudtRestrictions - приложение по работе с ограничениями на значения свойств, определяемых в терминах QUDT (запускается на Jboss)
4)thermal - приложение Рината, к которому добавлена функцинальность по связыванию по веществам (запускается на TomCat, после первого запуска необходимо прогнать скрипт по заполнению базы в папке scripts)
5)Jboss - сервер приложений, на котором работает qudtRestrictions. Чтобы все работало, нужно положить папку JBoss в корень диска C.
----
Файлы онтологии QUDT лежат в ресурсах проекта qudtRestrictions. Они непосредственно нигде не требуются, кроме этого приложения.
